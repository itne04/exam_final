🌐 ຕັ້ງຄ່າ Network & Static IP

ກ່ອນ Config ລະບົບ Windows Server ໃນ VMware ຕ້ອງຕັ້ງ Static IP ໃຫ້ຮຽບຮ้อยກ່ອນສະເໝີ

🔴 ເຮັດກ່ອນທຸກຢ່າງ! ເຊີເວີທີ່ເປັນ Domain Controller ຕ້ອງມີ IP ຄົງທີ່ ຫ້າມໃຊ້ DHCP ເດັດຂາດ — ຖ້າ IP ປ່ຽນ Client ຈະຊອກ AD ບໍ່ເຫັນ ລະບົບຈະເພພັງທັນທີ
1
ຕັ້ງຄ່າ VMware Network Adapter
VMware Settings
VM › Settings › Network Adapter › ✅ NAT
NAT Mode = VM ແຊຣ໌ Internet ຈາກເຄື່ອງ Host ໄດ້ ໂດຍບໍ່ຕ້ອງຕັ້ງຄ່າເພີ່ມເຕີມໃນເຄື່ອງ Host

ເປີດ Network Properties ເທິງ Windows Server
Windows

ເລືອກ "Use the following IP address" (Static — ຫ້າມເລືອກ Obtain automatically!)
3
ກວດເຊັກ Subnet ຕົວຈິງຂອງ VMware ກ່ອນໃສ່ IP
⚠️ ຕ້ອງເຮັດກ່ອນ

Subnet ຂອງ VMnet8 ຈະບອກວ່າ IP ໃນ VM ຕ້ອງຢູ່ກຸ່ມ (วง) ໃດ

📜 Group Policy Objects (GPO)

ແນວຄິດ: ຄືການອອກກົດໝາຍຈາກສ່ວນກາງ (Server) ເພື່ອບັງຄັບໃຊ້ກັບຄອມພິວເຕີ ຫຼື ພະນັກງານ (Users) ທີ່ຢູ່ໃນ OU ນັ້ນໆ

🧠 ຈື່ຫຼັກການເຮັດວຽກ 4 ຂັ້ນຕອນ:
1. ສ້າງໄຟລ໌ກົດໝາຍ (GPO) ຢູ່ພາຍໃຕ້ OU ທີ່ຕ້ອງການ
2. ເຂົ້າໄປແກ້ໄຂ (Edit) ເພື່ອເລືອກວ່າຈະສັ່ງບລັອກ ຫຼື ຕັ້ງຄ່າຫຍັງ
3. ກວດສອບການເຊື່ອມໂຍງ (Link) ວ່າກົດໝາຍຜູກກັບ OU ແລ້ວບໍ່
4. ສັ່ງອັບເດດ (gpupdate) ຢູ່ເຄື່ອງ Client ເພື່ອໃຫ້ກົດໝາຍມີຜົນທັນທີ
1
ເປີດເຄື່ອງມືຈັດການ Group Policy
ຂັ້ນຕອນທີ 1
Server Manager › Tools › Group Policy Management
ຄຳສັ່ງລັດ: ກົດປຸ່ມ Win + R ເທິງຄີບອດ → ພິມ gpmc.msc ແລ້ວກົດ Enter
2
ສ້າງກົດ GPO ໃໝ່ ຜູກເຂົ້າກັບ OU ເປົ້າໝາຍ
ຂັ້ນຕອນທີ 2
ເປັນຫຍັງຕ້ອງເຮັດຈຸດນີ້? ຖ້າເຈົ້າສ້າງ GPO ລ້າໆ ໂດຍບໍ່ຜູກ (Link) ກັບ OU, ກົດນັ້ນຈະບໍ່ມີຜົນກັບພະນັກງານຄົນໃດເລີຍ.
ຂະຫຍາຍ Forest › Domains › ChaleunIT.local › ຄລິກຂວາໃສ່ OU (ເຊັ່ນ: Sales_Dept) › Create a GPO in this domain, and Link it here...
ຕັ້ງຊື່ໃຫ້ຊັດເຈນໃນຊ່ອງ Name: ເຊັ່ນ "Block_CP_and_Wallpaper" ແລ້ວກົດ OK
3
ເຂົ້າໄປຕັ້ງຄ່າກົດບລັອກ (GPO Editor)
ຂັ້ນຕອນທີ 3
ຄລິກຂວາໃສ່ GPO ທີ່ຫາສ້າງມື້ກີ້ › Edit › ໜ້າຕ່າງ Group Policy Management Editor ຈະເປີດຂຶ້ນມາ
📍 ຕົວຢ່າງການຕັ້ງຄ່າບລັອກ Control Panel (ຫ້າມພະນັກງານຕັ້ງຄ່າເຄື່ອງ):
User Configuration › Policies › Administrative Templates › Control Panel › ດັບເບິລຄລິກ Prohibit access to Control Panel and PC settings › เลือก 🔘 Enabled › Apply → OK
📍 ຕົວຢ່າງການບັງຄັບຮູບພື້ນຫຼັງ (Desktop Wallpaper):
User Configuration › Policies › Administrative Templates › Desktop › Desktop › ດັບເບິລຄລິກ Desktop Wallpaper › ເລືອກ 🔘 Enabled
ໃນຊ່อง Wallpaper Name: ໃຫ້ໃສ່ Path ໄຟລ໌ຮູບ (ແນະນຳໃຫ້ໃຊ້ Local Path ຂອງເຄື່ອງ Client ຫຼື ໄຟລ໌ແຊຣ໌ ເຊັ່ນ C:\Windows\Web\Wallpaper\Windows\img0.jpg)
4
ວິທີແກ້ບັນຫາ Domain User ທຳມະດາ Login ຢູ່ໜ້າເຄື່ອງ Server ບໍ່ໄດ້
ຈຸດປາບຊຽນ
🔴 ງົງກັນຫຼາຍຈຸດນີ້: ເວລາເຈົ້າຢາກທົດສອບ GPO ໂດຍການ "Switch User" ຢູ່ໜ້າຈໍ VMware ຂອງເຄື່ອງ Server ເອງ, Windows Server ຈະບລັອກບໍ່ໃຫ້ User ທຳມະດາເຂົ້າ ເພາະມັນອັນຕະລາຍ. ຖ້າຢາກໃຫ້ Login ໄດ້ ຕ້ອງເປີດສິດກ່ອນ!
ໃນໜ້າ GPMC › ຄລິກໃສ່ໂຟນເດີ Domain Controllers › ຄລິກຂວາໃສ່ Default Domain Controllers Policy › Edit
Computer Configuration › Policies › Windows Settings › Security Settings › Local Policies › User Rights Assignment › ດັບເບິລຄລິກ Allow log on locally
ກົດປຸ່ມ Add User or Group... → ພິມຄຳວ່າ Domain Users → ກົດ Check Names (ມັນຈະຂີດກ້ອງ) → ກົດ OK → Apply → OK
5
ສັ່ງອັບເດດກົດ ແລະ ທົດສອບ (ລັນຄຳສັ່ງ)
ຂັ້ນຕອນສຸດທ້າຍ
ວິທີອັບເດດ: ໂດຍປົກກະຕິ Windows ຈະໃຊ້ເວລາ 90 ນາທີໃນການດຶງກົດ GPO ໃໝ່. ເຮົາຕ້ອງໃຊ້ຄຳສັ່ງລັດເພື່ອບັງຄັບໃຫ້ມັນດຶງກົດມາໃຊ້ທັນທີ.
# ເປີດ CMD (Command Prompt) ຂຶ້ນມາແລ້ວພິມ: gpupdate /force # ຖ້າທົດສອບຢູ່ເຄື່ອງ Server ໃຫ້ລັນຄຳສັ່ງນີ້ ແລ້ວເຮັດການ Sign Out ຫຼື Switch User # ຈາກນັ້ນ Login ດ້ວຍ User ທີ່ຢູ່ໃນ OU ທີ່ເຮົາຜູກກົດໄວ້ (ເຊັ່ນ: Sales_User)
ວິທີເບິ່ງຜົນຮັບ: ລອງຄລິກຂວາໃສ່ປຸ່ມ Start → ເລືອກ Control Panel ຫຼື Settings → ຖ້າກົດເຮັດວຽກ ມັນຈະຂຶ້ນເຕືອນທັນທີວ່າ "This operation has been cancelled due to restrictions..." ແປວ່າເຈົ້າເຮັດສຳເລັດແລ້ວ!

🔐 Share & NTFS Permissions

ແნວຄິດ: ການເຮັດລະບົບແຟ້ມຂໍ້ມູນສ່ວນກາງ (File Server) ທີ່ແບ່ງສິດໃຫ້ແຕ່ລະແຜນກເຂົ້າເຖິງໂຟນເດີບໍ່ຄືກັນ ໂດຍໃຊ້ລະບົບປະຕູປ້ອງກັນ 2 ຊັ້ນ

🚪 ປະຕູຊັ້ນທີ 1: Share Permission (ກວດສອບຕອນເຂົ້າມາທາງ Network)
ຕັ້ງຄ່າຢູ່ແທັບ Sharing → ໃຊ້ເພື່ອອະນຸຍາດໃຫ້ມອງເຫັນໂຟນເດີນີ້ຜ່ານເຄືອຂ່າຍໄຮ້ສາຍ/ມີສາຍໄດ້.

🔐 ປະຕູຊັ້ນທີ 2: NTFS Permission (ກວດສອບສິດໃນລະດັບຕົວຟາຍ/ໂຟນເດີແທ້ໆ)
ຕັ້ງຄ່າຢູ່ແທັບ Security → ໃຊ້ຄວບຄຸມລະອຽດ ໃຜອ່ານໄດ້ຢ່າງດຽວ ໃຜຂຽນໄດ້ ໃຜລຶບໄດ້.

🚀 File Server Resource Manager (FSRM)

ແნວຄິດ: ເປັນປລັກອິນເພີ່ມເຕີມເພື່ອເຮັດໃຫ້ File Server ຂອງເຮົາສະຫຼາດຂຶ້ນ ເຊັ່ນ: ຈຳກັດພື້ນທີ່ບໍ່ໃຫ້ພະນັກງານໃຊ້ເປືອງ, ບລັອກບໍ່ໃຫ້ເອົາເພງມາເກັບ, ແລະ ຊ່ອນໂຟນເດີທີ່ບໍ່ມີສິດເບິ່ງ

Quota Management (ຄຸມພື້ນທີ່ - ຫ້າມໃຊ້ເກີນ)
ບລັອກພື້ນທີ່
ຄວາມແຕກຕ່າງໃນຫ້ອງເສັງ (ອອກເລື້ອຍຫຼາຍ):
* 🔘 Hard Quota: ຖ້າຕັ້ງໄວ້ 100MB ພະນັກງານກັອບໄຟລ໌ຮອດ 100MB ລະບົບຈະ ບລັອກທັນທີ ກັອບເພີ່ມອີກບໍ່ໄດ້ເລີຍ.
* 🔘 Soft Quota: ຫາກກາຍ 100MB ລະບົບຈະ ບໍ່ບລັອກ ພະນັກງານຍັງໃສ່ໄຟລ໌ໄດ້ເລື້ອຍໆ ແຕ່ມັນຈະສົ່ງ Email ແຈ້ງເຕືອນໄປຫາ Admin ເທົ່ານັ້ນ.
ເປີດ FSRM ຈາກ Server Manager → Tools › Quota Management › Quotas › ຄລິກຂວາໃສ່ພື້ນທີ່ວ່າງ → Create Quota...
ຊ່ອງ Quota path: ກົດ Browse ເລືອກໂຟນເດີທີ່ຕ້ອງການຈຳກັດ ເຊັ່ນ C:\Company_Data
ເລືອກ 🔘 Define custom quota properties → ກົດ Custom Properties...
ໃສ່ເລກກຳນົດພື້ນທີ່ ເຊັ່ນ 100 ແລ້ວປ່ຽນຫົວໜ່ວຍເປັນ MB
ເລືອກ 🔘 Hard quota → ກົດ OK → ກົດ Create → ມັນຈະຖາມໃຫ້ເຊັບ Template ໃຫ້ເລືອກ Save without template ແລ້ວກົດ OK

File Screening (ບລັອກປະເພດໄຟລ໌ - ຫ້າມເກັບເພງ/ໜັງ)

ວິທີທົດສອບ: ໄປທີ່ເຄື່ອງ Client ເຂົ້າໄປໃນໂຟນເດີແຊຣ໌ ແລ້ວລອງສ້າງໄຟລ໌ເປົ່າໆ ແລ້ວປ່ຽນຊື່ທ້າຍເປັນ song.mp3 → ມັນຈະຂຶ້ນ Error ບອກວ່າເຈົ້າບໍ່ມີສິດວາງໄຟລ໌ນີ້!

Access-Based Enumeration alebo ABE (ຊ່ອນໂຟນເດີລັບ)
ຄຸນສົມບັດ ABE
🔴 ຈຸດນີ້ຄົນມັກຫຼົງ: ປົກກະຕິແລ້ວ ຖ້າເຮົາມີໂຟນເດີຍ່ອຍຊື່ວ່າ "HR_Data" ແລະ "Sales_Data" ຢູ່ທາງໃນ. ເຖິງວ່າພະນັກງານ Sales ຈະບໍ່ມີສິດເຂົ້າໂຟນເດີ HR ແຕ່ພວກເຂົາກໍຍັງ "ມອງເຫັນຮູບໄອຄອນໂຟນເດີ HR" ຕັ້ງຢູ່ (ແຄ່ດັບເບິລຄລິກເຂົ້າໄປແລ້ວຂຶ້ນ Access Denied).

ຖ້າເຮົາເປີດໃຊ້ງານ ABE → Windows ຈະກວດສອບວ່າ ໃຜບໍ່ມີສິດເຂົ້າໂຟນເດີ HR ມັນຈະ "ຊ່ອນຮູບໂຟນເດີນັ້ນອອກໄປເລີຍ" ເຮັດໃຫ້ພະນັກງານ Sales ບໍ່ຮູ້ວ່າມີໂຟນເດີ HR ຢູ່ເທິງໂລກນີ້!
ໄປທີ່ Server Manager › ເລືອກເມນູ File and Storage Services ເບື້ອງຊ້າຍ › ຄລິກໃສ່ເມນູ Shares › ຄລິກຂວາໃສ່ຊື່ໂຟນເດີແຊຣ໌ຂອງເຮົາ (Company_Data) → ເລືອກ Properties
ໄປທີ່ແທັບ Settings › ຕິກເລືອກ 🔲 Enable access-based enumeration › ກົດ Apply → OK

Shadow Copies (ລະບົບກູ້ຄືນໄຟລ໌ເວລາພະນັກງານຫຼົງລຶບ)
ລະບົບກູ້ຄືນຂໍ້ມູນ
ສະຖານະການຕົວຈິງ: ພະນັກງານເຮັດວຽກໄປແລ້ວຫຼົງມືລຶບໄຟລ໌ສຳຄັນຖິ້ມ ຫຼື ແກ້ໄຂຂໍ້ມູນຜິດພາດແລ້ວຢາກໄດ້ເວີຊັນຂອງມື້ວານນີ້ຄືນ. Admin ບໍ່ຕ້ອງໄປກູ້ໃນຖັງຂີ້ເຫຍື້ອ ໃຫ້ໃຊ້ Shadow Copies.
ເປີດ This PC ຢູ່ເຄື່ອງ Server › ຄລິກຂວາໃສ່ Drive C: › ເລືອກ Configure Shadow Copies...
ຄລິກເລືອກ Volume 🔘 C: → ກົດປຸ່ມ Enable → ກົດປຸ່ມ Create Now (ເພື່ອທົດສອບຖ່າຍ Snapshot ເກັບໄວ້ຕອນນີ້ທັນທີ) → ກົດ OK
ວິທີກູ້ຄືນ (ເຮັດຢູ່ເຄື່ອງ Client ຫຼື Server ກໍໄດ້): ຄລິກຂວາໃສ່ໂຟນເດີ Company_Data → ເລືອກ Properties → ໄປທີ່ແທັບ "Previous Versions" → ເລືອກວັນເວລາທີ່ຢາກຍ້ອນກັບໄປ → ກົດປຸ່ມ Restore... ໄຟລ໌ທີ່ເຄີຍຖືກລຶບຈະກັບຄືນມາຄືເກົ່າຢ່າງມະຫັດສະຈັນ!
